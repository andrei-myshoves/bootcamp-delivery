## Git-flow и деплой

```
feature/*  →  dev  →  main
              ↓       ↓
         dev.*.tech   prod домен
```

### Ветки

- **`dev`** — текущая разработка фич
- **`main`** — прод

### Цикл одной фичи

1. Отвести ветку от **`dev`**
2. Разработать локально
3. Pull Request → **`dev`**
4. После merge — GitHub Actions деплоит на стенд, например `dev.delivery.andreimushavets.tech`
5. Проверить фичу на dev-стенде
6. Pull Request **`dev` → `main`**
7. После merge — деплой на прод, например `delivery.andreimushavets.tech`
8. **Ветку `dev` не удалять** — она нужна для следующих спринтов

## Требования к качеству

| Что          | Как                   |
| ------------ | --------------------- |
| Компоненты   | Storybook             |
| Компоненты   | React Testing Library |
| Утилиты      | Jest (unit)           |
| Фича целиком | Playwright (e2e)      |

Тесты можно вынести в отдельный PR после фичи, но **в `main` без тестов не мержим** — в прод только с покрытием.
